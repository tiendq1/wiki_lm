# Stable Memory

## Người dùng

- Frontend web developer, sinh năm 1988.
- Mạnh và thích UI/CSS.
- Công việc chính hiện tại: React Native, cross-platform mobile.
- Thỉnh thoảng làm React web.
- Công việc chính hiện tại trong mảng nghiệp vụ: vay cầm cố bằng sổ tiết kiệm.

## Stack thường gặp

- React Native: `0.77.0`.
- React web: JavaScript và TypeScript.
- Mobile navigation: `react-navigation`.
- Web routing: `react-router-dom`, `react-router`, version v5 hoặc v6 tùy project.
- State management: Redux Toolkit, React Context.
- Styling: Tailwind CSS, ưu tiên atomic CSS style.

- Ngắn gọn.

- Không vòng vo, không cả nể.
- Không giải thích lại frontend căn bản nếu không cần.

## Rule vận hành

- Khi cần scan project, đọc file, dò bug, chạy command để mò fixbug: agent được phép làm.
- Nếu có rủi ro security, secret, token, credential, dữ liệu nhạy cảm, network, upload, external API, hoặc rò rỉ thông tin ra ngoài: hỏi confirm trước.
- Nếu là fix bug, ghi command quan trọng vào `terminal/fixbug.md`.
- Nếu là lỗi mobile, ghi vào `errors/mobile/`.
- Sau khi xử lý việc quan trọng, cập nhật `daily-report/`.

## Mục tiêu dài hạn

- Xây dựng agent thành trợ lí thân cận.
- Người dùng cung cấp data hằng ngày để agent hiểu ngữ cảnh và tư vấn chính xác hơn.
- Mục tiêu xa hơn: tiến tới kiểu engineering AI, nơi người dùng đưa ý tưởng, agent hiểu, chia nhỏ, làm dự án, và cập nhật project dần theo yêu cầu.
